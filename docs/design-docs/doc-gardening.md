# Doc Gardening

## Purpose

Continuously remove stale instructions and keep documentation aligned with code.

## Cadence

- Review high-impact docs regularly.
- Update docs in the same change where behavior changes.

## What to Prune

- stale references to removed modules
- contradictory workflow instructions
- overly generic rules with no actionable value

## What to Add

- new architectural boundaries
- new domain concepts or lifecycle states
- new operational runbooks or failure handling notes

## Checklist

1. Link from AGENTS or relevant index docs.
2. Ensure wording is specific and testable.
3. Avoid bloating root entry-point files.
