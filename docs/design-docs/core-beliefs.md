# Core Beliefs

## Agent-Legible First

If knowledge is not in the repository, it is effectively unavailable for reliable agent execution.

## Progressive Disclosure

Start from a small stable map, then follow links to deeper context only when needed.

## Enforce Invariants, Not Style Micromanagement

Encode boundaries and correctness constraints so implementations can vary without architecture drift.

## Small, Verifiable Iterations

Prefer short feedback loops with explicit verification to reduce regression risk.

## Documentation As Runtime Support

Docs are operational assets. Keep them concise, discoverable, and synchronized with code behavior.
